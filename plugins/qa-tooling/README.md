# qa-tooling — Claude Code Plugin

QA/QC workflow automation for web projects. Auto-detects your stack, generates a `qa-profile.json` for the [QA Annotator Chrome extension](../README.md), and syncs QA reports into Jira via MCP.

**Status:** Phase 1 (MVP) — four commands, four skills.

---

## Interaction convention

**Commands and skills in this plugin must use `AskUserQuestion` for every user prompt** — never inline `? ...` chat prompts. This applies to confirmations, choices, and clarifications. If a value can be inferred from detection, the profile, or git context, do not ask at all. When new commands are added, include explicit option sets in the question call (1–3 questions per call, `header` ≤ 12 chars).

---

## Commands

| Command | Purpose |
|---|---|
| `/qa:init` | Auto-detect framework/locales/sections/Figma/git, generate `docs/qa/qa-profile.json` and a README. Asks at most three questions (Jira project, parent ticket, Figma key). |
| `/qa:doctor` | Health-check: profile validity, source conventions, `.gitignore`, Figma tree freshness, MCP server availability. |
| `/qa:sync` | Read latest QA report → create one Jira sub-task per issue (or `--mode tasks` / `--mode append`). Idempotent — safe to re-run. |
| `/qa:figma-sync` | Crawl the Figma file via MCP and write `docs/qa/qa-figma-tree.json` so the extension can auto-fill nearest-frame deeplinks for picked elements. Run once, re-run when designer reorganizes the file. |

| File | Generated by | Read by |
|---|---|---|
| `docs/qa/qa-profile.json` | `/qa:init` | QA Annotator extension (Settings → Import) |

| `docs/qa/README.md` | `/qa:init` | Devs, QA — quick reference |

## Install

The plugin is published as part of the **qa-annotator** Claude Code marketplace. In a Claude Code session at any project:

```
/plugin marketplace add dongquoctien/qa-annotator-extension
/plugin install qa-tooling@qa-annotator
```

Then run `/qa:init` to bootstrap a project. Subsequent updates pull automatically — re-run `/plugin marketplace update qa-annotator` to refresh.

> Manual install (no marketplace): clone this repo and add an absolute path to the project's `.claude/settings.json`:
> ```jsonc
> { "plugins": ["<absolute-path>/qa-annotator-extension/plugins/qa-tooling"] }
> ```

## Recommended MCP servers

Not required to install — `/qa:doctor` warns if any are missing.

| MCP | Used for |
|---|---|
| `mcp-atlassian` | `/qa:sync` (Jira create/update) |
| `figma` | `/qa:figma-sync` (build per-element Figma deeplink tree) |
